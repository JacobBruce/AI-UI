Install Guide

Download the latest release of AI UI then follow the instructions for your platform:

Click to show install instructions for Windows

First you will need to install Python:

 • Python (3.12.x recommended)

CUDA is optional but highly recommended if you have an Nvidia GPU:

 • CUDA (12.4 recommended)

Now open a command prompt as an administrator and create a virtual Python environment using this command:

python -m venv C:/venv

This will create a folder called venv on the C drive containing the Python environment. Replace C:/venv with something else to change the location of the virtual environment.

Now you can activate the virtual environment by running the activate.bat file like this:

C:/venv/Scripts/activate.bat

Now install the required Python packages into the virtual environment using this command:

pip install -r C:/AI_UI/engine/requirements.txt

Replace C:/AI_UI/ with the location where AI UI was extracted (should contain AI_UI.exe)

If your GPU supports CUDA then run this command before installing the requirements in the previous step:

pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu124
Click to show install instructions for Linux

First ensure you have Python installed (3.12.x recommended). You may also need to install pip and the tool for creating virtual Python environments using these commands:

sudo apt install python3-pip
sudo apt install python3.12-venv

Now open a terminal and navigate to the directory where you want to create a virtual Python environment then run this command:

python3 -m venv ./venv

This will create a folder called venv containing the Python environment.

Now you can activate the virtual environment by running this command:

source ./venv/bin/activate

Now install the required Python packages into the virtual environment using this command:

pip3 install -r ./AI_UI/engine/requirements.txt

Replace ./AI_UI/ with the location where AI UI was extracted (should contain electron)

If your GPU supports CUDA then run this command before installing the requirements in the previous step:

pip3 install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu124

Now exit the virtual environment and run this command to install the nodejs package manager:

sudo apt install npm

Now navigate to the AI UI app folder then install the required nodejs packages using these commands:

cd AI_UI/resources/app
npm install

Now navigate back to the AI_UI folder (cd ../..) and launch AI UI using this command:

./electron --no-sandbox

Important Info

Windows Users

If you place AI UI into your Program Files directory or any other protected directory then you will need to run the app in administrator mode. It is recommended to place AI UI in another location (such as C:/AI_UI/) to avoid the use of administrator mode.

Linux Users

Before starting AI UI from a terminal, make sure you navigate to the location where the electron binary is located instead of trying to launch it from another directory. If you don't do this AI UI will use the wrong working directory and it will fail to read/write files.

Getting Started

After all the requirements have been installed you can launch the AI UI app. To get started you will need to visit the Settings tab so the engine knows where to find things like the Python environment and your model files.

In this example the 'Python Binary' setting would be C:/venv/Scripts/python.exe for Windows or path/to/venv/lib/python3 for Linux. Check the Console tab for errors if the engine wont start.

To make the chat bot work you'll need to download a text generation AI model from Hugging Face (must use the HF Transformers format). Models which have been fine-tuned on conversational text should work best for the chat bot.

If you want to generate images or want the chat bot to send messages with images you will also need to download an image generation AI model which uses the HF Diffusers format, many can be found on Hugging Face.


A list of all the setup arguments which can be used for the Model Setup on the Settings tab.

torch_dtype (default: auto)

Description: The Torch data type used by the text generation model. Models using float16 will be faster and use less memory but are slightly lower quality than float32 models.

Valid Values: float32, float16, bfloat16

torch_compile (default: false)

Description: Enable to make PyTorch code run faster by using JIT-compiling. This can greatly increase generation speeds for some models. This feature may not be available on Windows platforms.

Valid Values: true, false

reserve_vram_mb (default: 0)

Description: Reserve VRAM to prevent large text generation models from filling it up and leaving no room for other models. The default value increases to 3000 if using an image generation model. This only has an effect when the Compute Device is set to Auto.

Valid Values: any integer (megabytes)

min_res_vram_mb (default: 800)

Description: The minimum amount of VRAM to keep reserved for critical tasks and small models which get loaded and unloaded on the fly (such as TTS models and face animation models). This only has an effect when the Compute Device is set to Auto.

Valid Values: any integer (megabytes)

use_safetensors (default: none)

Description: If this argument isn't set the text generation model will try loading from a .safetensors file if one is available. If true the model will be forcibly loaded from a .safetensors file. If set to false the model will not try loading from safetensors.

Valid Values: true, false

custom_model_code (default: false)

Description: This allows the text generation model to use custom code. Technically it sets the trust_remote_code argument to true or false. Enable only with trusted models.

Valid Values: true, false

custom_token_code (default: false)

Description: This allows the text tokenizer to use custom code. Technically it sets the trust_remote_code argument to true or false. Enable only with trusted tokenizers.

Valid Values: true, false

load_in_8bit (default: false)

Description: Load a 8-bit text generation model quantized with GPTQ. Quantized models typically use much less memory and run faster but produce lower quality results.

Valid Values: true, false

use_chat_template (default: false)

Description: Some models can automatically format the chat log into the template they use. This requires certain Roles to be set, usually 'user' and 'assistant'. If the 'system' role is unsupported set Prompt Persistence to 'No Prompt' in the AI Settings of the chat bot.

Valid Values: true, false

use_tool_template (default: false)

Description: Enables a specialized chat template for tool use models. Not all tool use models support this feature. If this doesn't work, try enabling use_chat_template with this option disabled, and provide tool details in the system prompt. The system prompt isn't required when this option is enabled. Edit /engine/tools/tool_funcs.py to change the tool functions.

Valid Values: true, false

skip_special_tokens (default: true)

Description: If enabled the tokenizer will skip special tokens when decoding the output of the text generation model. Should usually be enabled.

Valid Values: true, false

cleanup_token_space (default: true)

Description: Enable to cleanup any spaces that were added when splitting the input text during the tokenization process. Should usually be enabled.

Valid Values: true, false

speech_recog_class (default: 'AutoModelForSpeechSeq2Seq')

Description: The transformers model class used for the speech recognition model (aka text-to-speech model). This setting should only be changed if you know what you are doing.

Valid Values: class name

text_lang_class (default: 'AutoModelForCausalLM')

Description: The transformers model class used for the text generation model. This only has an effect when Text Model Type is set to 'Auto Language Model' on the Settings tab.

Valid Values: class name

multi_modal_class (default: 'AutoModelForImageTextToText')

Description: The transformers model class used for the text generation model. This only has an effect when Text Model Type is set to 'Auto Multimodal Model' on the Settings tab.

Valid Values: class name

process_vision_func (default: 'qwen_vl_utils.process_vision_info')

Description: The function used by multimodal models to process vision data such as images and videos. Must be a module name and a function name separated by a '.' character (e.g. my_module.my_func).

Valid Values: class name

attn_implementation (default: 'sdpa')

Description: The attention implementation to use in the text generation model (if relevant). If available, SDPA will be used be default, otherwise the manual 'eager' implementation will be used.

Valid Values: 'eager', 'sdpa', 'flash_attention_2'

roles_append_str (default: ': ')

Description: The default chat log format appends a colon and a space after the usernames/roles (e.g. Human: hello). Use this to change the appended text. Use \n for new lines (e.g. roles_append_str=\n). This only has an effect when use_chat_template and use_tool_template are set to false.

Valid Values: any string

vocoder_model (default: none)

Description: The directory or Model ID of a speech vocoder model to be used by the text-to-speech model. Usually not needed and may not be supported by some TTS models.

Valid Values: any string (vocoder model folder)

tokenizer_model (default: none)

Description: The directory or Model ID of a tokenizer/processor for the text generation model. Most text generation models should come with tokenizers so this usually isn't needed.

Valid Values: any string (tokenizer model folder)

model_adapter (default: none)

Description: The directory or Model ID of an adapter for the text generation model. Adapters adjust the base model so it acts like a fine-tuned model without the need to download a full fine-tuned model.

Valid Values: any string (adapter model folder)

im_vae_file (default: none)

Description: The path to a Variational Auto-Encoder (VAE) model file to be used by the image generation model. Cannot be directory, must be path to a single model file.

Valid Values: any string (vae model file)

im_config_file (default: none)

Description: The path to a .yaml config file which specifies the architecture for the image generation model. This only has an effect when loading single model files, can be useful for offline model loading.

Valid Values: any string (model config file)

im_lora_file (default: none)

Description: The file name (not a full path) of a LoRA model to be used as an adapter for the image generation model (e.g. pixel-art-xl.safetensors). Requires im_lora_dir to be set.

Valid Values: any string (lora model file)

im_lora_dir (default: none)

Description: The directory containing the LoRA adapter for the image generation model. This can also point to an online Hugging Face repository (e.g. nerijs/pixel-art-xl)

Valid Values: any string (lora model directory)

im_lora_scale (default: 1.0)

Description: Controls how much the LoRA adapter will influence the output of the image model. Higher values will give the LoRA more influence on the output. Should be a value between 0 and 2.

Valid Values: decimal number (0.0 to 2.0)

im_torch_dtype (default: float16)

Description: The Torch data type used by the image generation model. Models using float16 will be faster and use less memory but are slightly lower quality than float32 models.

Valid Values: float32, float16

im_safety_check (default: true)

Description: Enable the safety checker to censor NSFW images produced by the image generation model. Set to false to disable the safety checker and prevent it from producing blank images.

Valid Values: true, false

im_cpu_offload (default: true)

Description: Enable CPU offloading for the image generation model to lower VRAM usage. Submodules of the model will be offloaded from the GPU to system memory when not in use. May reduce generation speed.

Valid Values: true, false

im_att_slicing (default: true)

Description: Enable attention slicing for the image generation model to lower VRAM usage. Splits the attention computations into steps instead of doing them all at once. The default value is false when using Stable Diffusion XL as it can adversely impact generation speed.

Valid Values: true, false

im_from_single_file (default: false)

Description: Force the image generation model to load from a single file such as a .safetensors or .ckpt file. Can be a local file path or a link to an online file (e.g. https://mysite.com/mymodel.safetensors)

Valid Values: true, false

im_local_files (default: false)

Description: Force the image generation model to load from local files only. This may be necessary to avoid errors when loading local models. Technically it sets the local_files_only argument to true or false.

Valid Values: true, false

im_use_safetensors (default: none)

Description: If this argument isn't set the image generation model will try loading from a .safetensors file if one is available. If true the model will be forcibly loaded from a .safetensors file. If set to false the model will not try loading from safetensors.

Valid Values: true, false
